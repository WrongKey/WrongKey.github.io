title: AWS系列之IAM

最近一段时间一直在学习AWS相关的东西，近两周把IAM（Identity and Access Management）的文档过了一遍, 现在总结一下学到的东西。

##理论

###IAM定义：

其实，这就和Linux的权限系统很相似，只是将对文件系统的操作换成了对AWS上的资源的操作。我们在注册成为AWS用户时所使用的账户就是就是我们根账户， 根账户就如同Linux上的root用户，对该用户在AWS上的资源拥有所有的操作权限。我们一般不建议使用跟账户去对AWS上的资源进行操作。

###IAM能够做什么
试想一下，有这样一个组织，这个组织当中的所有人都使用同一个AWS账户下的资源，并且都使用根用户对AWS上的资源进行操作，这样会面临什么样的问题呢？

其实这种做法带来的最直接也是最严重的问题就是安全方面的问题，数据的安全，应用服务器的安全，这些安全问题可能来自于错误的操作，也可能来自于恶意的破坏。

让我们来看看IAM为我们提供了哪些功能：

  • 对你 AWS 账户的共享访问权限

    可以向其他人员授予管理和使用您 AWS 账户中的资源的权限，而不必共享你的密码或访问密钥。

  • 精细化权限

    可以针对不同资源向不同人员授予不同权限。

  • 与很多的AWS服务集成

    可以控制 Amazon EC2 上运行的应用程序针对 AWS 资源的安全访问权限。

很重要的一点是IAM服务是作为你AWS账户的一个可以免费使用的功能。

你可以通过AWS 管理控制台（UI），命令行工具（aws cli）, 软件开发工具包(sdk 编程式)以及HTTPS API 使用IAM服务。
